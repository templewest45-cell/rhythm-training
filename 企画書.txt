# アプリ概要

カメラやマイクを活用し、「リズムを感じる」「動きをまねする」「音楽に合わせて身体を動かす」活動を支援するWebアプリを作成する。

手拍子だけでなく、足踏み、手を振る、身体を揺らす、打楽器演奏、指揮動作など、多様な身体表現に対応できる構成にする。

対象は特別支援学校の小学部・中学部・高等部の児童生徒を想定する。低年齢向けに限定せず、年齢よりも実態に応じて使える教材アプリとする。

# 開発のねらい

特別支援学校の音楽活動において、リズムを維持する、タイミングを合わせる、動きを模倣する、音楽に合わせて身体を動かすことを支援する。

本アプリでは、「見る」「聞く」「動く」を組み合わせ、リズムやタイミングを視覚的・身体的に理解しやすくすることを目指す。

# 実装するモード

## 1. 基礎リズムモード
光る位置やリズム表示を見ながら、リズムに合わせて手拍子・足踏み・タップ・打楽器などを行う。

目的：
- 拍感
- 一定テンポ
- 待つ
- リズム理解

仕様：
- 4拍または8拍のリズムパターンを表示する
- 現在の拍が光る
- ユーザーの入力タイミングを判定する
- 成功時は肯定的なフィードバックを返す

## 2. まねっこリズムモード
キャラクターやモデル動作を見て、同じ動きを真似する。

動きの例：
- 手拍子
- 足踏み
- 手を振る
- 膝タッチ
- 身体表現

目的：
- 模倣
- 注視
- タイミング合わせ
- 身体表現

仕様：
- キャラクターがリズムに合わせて動く
- ユーザーはそれを見て真似する
- カメラまたはマイクで反応を判定する
- 最初は厳密な姿勢判定ではなく、「動いたタイミング」の判定を優先する

## 3. きいてまねモード
音だけを聞いてリズムを再現する。

目的：
- 聴覚模倣
- 記憶
- 注意保持
- リズム再現

仕様：
- お手本リズムを再生する
- 譜面や視覚ガイドは表示しない、または設定で非表示にできる
- 再生後、ユーザーが同じリズムを打つ
- 入力タイミングをお手本と比較して判定する

## 4. 音楽あそびモード
曲に合わせて自由にリズム活動を行う。

内容：
- 手拍子
- 足踏み
- 身体表現
- ダンス風動作

目的：
- 音楽への参加
- リズムへの親しみ
- 身体活動
- 表現活動

仕様：
- 厳密判定よりも「のれている」「参加できている」ことを重視する
- 成功体験を重視する
- リザルトもやわらかい表現にする

## 5. 指揮者モード
音楽に合わせて腕を振り、最後に止める。

目的：
- テンポ感
- 抑制
- 切り替え
- 身体コントロール

仕様：
- カメラで腕の動きや身体の動きを検出する
- 一定テンポで振れているかを見る
- 最後の「止め」で静止できたかを見る
- 指揮者・オーケストラ風の演出を入れる

## 6. 楽器モード
様々な入力方法でリズム活動を行う。

対応想定：
- タンバリン
- 太鼓
- 机
- 足踏み
- 手拍子

仕様：
- マイク入力を中心に、音のピークを検出する
- 音が出る教材・楽器と組み合わせて使えるようにする
- グループ活動でも使いやすい画面にする

## 7. フリー作成モード（教師用）
教師がオリジナル問題を作成できる。

設定項目：
- BPM
- 拍数
- リズムパターン
- 動きの種類
- 曲
- 判定難易度

仕様：
- 教師が授業や児童生徒の実態に合わせてリズム課題を作れるようにする
- 保存・読み込みができるとよい
- 最初はローカル保存でよい

# 判定システム

## マイク判定
Web Audio APIを使い、手拍子や打楽器音のピークを検出する。

## カメラ判定
カメラ映像から身体の動きを検出する。
初期実装では高度な姿勢推定よりも、フレーム差分などによる「動きの大きさ」の検出を優先する。
発展としてMediaPipe Pose等の利用も検討する。

## 複合判定
モードに応じて以下を選べるようにする。

- マイクのみ
- カメラのみ
- マイク＋カメラ

マイク＋カメラでは、最初は「どちらかが反応すればOK」のやさしい判定を基本とする。

# フィードバック設計

「Perfect」「Miss」のような強い失敗表現に偏らず、以下のような肯定的な表現を使う。

- ぴったり！
- いいリズム！
- のれてる！
- おしい！
- もういっかい！

リザルトでは、得点だけでなく以下を表示する。

- 合っていた回数
- 少し早かった回数
- 少し遅かった回数
- もう一度やるボタン

# UI方針

- シンプル
- 分かりやすい
- 見通しが持てる
- 次の動きが分かる
- 過度に幼いデザインにしない
- 音楽活動らしさを大切にする

演出例：
- 光る
- 揺れる
- 拍手
- キャラクターリアクション
- エフェクト表示

# 技術構成

- React
- TypeScript
- Web Audio API
- getUserMedia
- 必要に応じてTone.js
- 必要に応じてMediaPipe Pose
- Cloudflare Pagesで公開できる構成

# 最初に作るMVP

まずは以下を優先して実装する。

1. 基礎リズムモード
2. マイク入力による手拍子判定
3. カメラ入力による動き検出
4. リズムパターン表示
5. 現在の拍が光るUI
6. 判定結果の表示
7. 教師用の簡易設定画面

# 注意点

- 児童生徒の年齢を低学年に限定しない
- 特別支援学校の音楽授業での使用を想定する
- 正確さだけでなく、音楽への参加や成功体験を重視する
- 判定は厳しすぎない
- 1プレイは短く、終わりやすくする
- 教師が実態に合わせて調整できるようにする

この仕様をもとに、まずはMVPとして動作するWebアプリを作成してください。
